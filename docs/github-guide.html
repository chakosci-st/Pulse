PULSE GitHub Workflow Guide

Copy-paste Git and GitHub snippets for your C# MVC multi-project repository

Replace <your-user-or-org> and branch names with your real values before using the commands.

Recommended Branch Strategy

Use a clean and simple workflow for development, bug fixing, and releases.

  • main → production-ready code
  • develop → integration branch for ongoing work
  • feature/<name> → new features
  • bugfix/<name> → normal bug fixes
  • hotfix/<name> → urgent production fixes
Initial setup

Initialize and push repository

git init
git add .
git commit -m "chore: initial commit"
git branch -M main
git remote add origin https://github.com/<your-user-or-org>/pulse.git
git push -u origin main
Develop branch

Create and push develop branch

git checkout -b develop
git push -u origin develop
Feature

Start a new feature

git checkout develop
git pull origin develop
git checkout -b feature/customer-search

git add .
git commit -m "feat: add customer search"
git push -u origin feature/customer-search
Bug fix

Start a bugfix branch

git checkout develop
git pull origin develop
git checkout -b bugfix/login-validation

git add .
git commit -m "fix: correct login validation logic"
git push -u origin bugfix/login-validation
Production fix

Create a hotfix from main

git checkout main
git pull origin main
git checkout -b hotfix/session-timeout-fix

git add .
git commit -m "fix: resolve session timeout issue"
git push -u origin hotfix/session-timeout-fix
Merge

Merge feature into develop locally

git checkout develop
git pull origin develop
git merge feature/customer-search
git push origin develop
Delete branch

Delete branch after merge

git branch -d feature/customer-search
git push origin --delete feature/customer-search
Release

Create release tag

git checkout main
git pull origin main
git tag -a v1.0.0 -m "Initial release"
git push origin v1.0.0
Sync branch

Update feature branch from develop

git checkout develop
git pull origin develop
git checkout feature/customer-search
git merge develop
git push origin feature/customer-search
Conflict resolution

Resolve merge conflict

git status

# Edit conflicted files manually

git add .
git commit -m "fix: resolve merge conflict with develop"
git push origin feature/customer-search
Revert

Revert a bad commit

git revert <commit-hash>
git push origin develop
Clone

Clone repository

git clone https://github.com/<your-user-or-org>/pulse.git
cd pulse
git checkout develop

Basic GitHub Actions CI

Save this as .github/workflows/ci.yml

name: CI

on:
  push:
    branches:
      - main
      - develop
      - feature/**
      - bugfix/**
      - hotfix/**
  pull_request:
    branches:
      - main
      - develop

jobs:
  build-and-test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout source
        uses: actions/checkout@v4

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: 8.0.x

      - name: Restore dependencies
        run: dotnet restore ./src/Pulse.sln

      - name: Build
        run: dotnet build ./src/Pulse.sln --configuration Release --no-restore

      - name: Test
        run: dotnet test ./src/Pulse.sln --configuration Release --no-build
